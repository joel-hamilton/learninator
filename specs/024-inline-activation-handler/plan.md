## Summary

Inline the `handleActivation()` function from `src/shared/activate-mission.ts` into its 5 call sites, then delete the module. The function wraps only 3 lines of real logic ‚Äî a conditional check, an async title generation call, and a redirect header set ‚Äî but adds a module boundary (import/export, separate file) that obscures the flow. After inlining, remove the import line and the `handleActivation` reference from each file, then delete `src/shared/activate-mission.ts`.

### Inline Plan (per call site)

Each call site currently has this pattern:

```typescript
const activated = await handleActivation(result, missionId, missionChatService, c);
if (activated) return activated;
```

This gets replaced with inline code equivalent to `handleActivation`'s body:

```typescript
if (result.didActivate) {
  await missionChatService.generateTitle(missionId);
  c.header("HX-Redirect", `/missions/${missionId}`);
  return c.body(null);
}
```

The 5 call sites are:

1. `src/routes/missions.ts` line 97 ‚Äî `POST /` (create mission from first chat message)
2. `src/routes/missions.ts` line 343 ‚Äî `POST /:missionId/chat` (chat message handler)
3. `src/routes/onboarding.ts` line 39 ‚Äî `POST /:missionId/guided/start`
4. `src/routes/onboarding.ts` line 94 ‚Äî `POST /:missionId/guided/answer`
5. `src/routes/onboarding.ts` line 140 ‚Äî `POST /:missionId/guided/skip`

After all 5 sites are updated:
- Remove `import { handleActivation } from "../shared/activate-mission.js";` from `src/routes/missions.ts` (line 15)
- Remove `import { handleActivation } from "../shared/activate-mission.js";` from `src/routes/onboarding.ts` (line 7)
- Delete `src/shared/activate-mission.ts`
- Verify no remaining imports from `activate-mission` in the codebase
