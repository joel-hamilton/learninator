## Phase 0: Research

**Preamble**: This feature needs no external research. The 13 functions being refactored are fully visible in a single file (`src/views/fragments.ts`). The differences between them are:

| Dimension | Next lesson | Regeneration | Bridging |
|-----------|-------------|--------------|----------|
| Badge text (polling/running) | "Generating" | "Regenerating" | "Generating" |
| Header text (polling/running) | "Creating your next lesson…" / "Creating sub-lesson…" | "Rewriting lesson at new difficulty…" | "Creating bridging lesson…" |
| Status polling suffix | `generate-next` / `generate-sub-lesson` | `regenerate` | `generate-bridging` |
| Link color (done) | `var(--rubric)` | `var(--accent)` | `var(--accent)` |
| Done badge text | "Ready" | "Updated" | "Ready" |
| Done link text | "Start Lesson X: title →" | "View Lesson X: title →" | "Start Lesson X: title →" |
| Error prefix | "Failed to generate next lesson: " | "Failed to regenerate lesson: " | "Failed to create bridging lesson: " |
| Missing bar | Applicable (for "not_found" in renderJobStatus) | Falls through to error | Falls through to error |
| isSub param | Polling/running accept it | Not needed | Not needed |

### Consolidation Strategy

The `GenStyle` config object will capture:
- `badgeText`: the badge label for polling/running states ("Generating", "Regenerating")
- `headerText`: the header string for polling/running states
- `statusSuffix`: the URL path suffix for status polling
- `linkColor`: CSS color variable for done bar link
- `doneBadgeText`: badge text for done state
- `doneLinkPrefix`: prefix for the done bar link ("Start Lesson", "View Lesson")
- `errorPrefix`: error message prefix
- `supportsSub`: boolean indicating whether isSub parameter is accepted
- `hasMissingBar`: boolean indicating whether missing state is supported (next only)

The `JobStatus` type will be a string union: `"polling" | "running" | "done" | "error" | "missing"`.

## Phase 1: Design

### Data Model

This feature introduces a configuration type, not a database entity:

```typescript
type LessonInfo = {
  number: number;
  subNumber: number | null;
  title: string;
};

type JobStatus = "polling" | "running" | "done" | "error" | "missing";

interface GenStyle {
  badgeText: string;         // e.g. "Generating", "Regenerating"
  headerText: string;        // e.g. "Creating your next lesson…"
  statusSuffix: string;      // e.g. "generate-next", "regenerate"
  linkColor: string;         // CSS variable, e.g. "var(--rubric)", "var(--accent)"
  doneBadgeText: string;     // e.g. "Ready", "Updated"
  doneLinkPrefix: string;    // e.g. "Start Lesson", "View Lesson"
  errorPrefix: string;       // e.g. "Failed to generate next lesson: "
  supportsSub: boolean;      // Does polling/running accept isSub?
  hasMissingBar: boolean;    // Does the type support the "missing" state?
}
```

Three config instances:
- `nextStyle` — standard next-lesson generation
- `regenStyle` — lesson regeneration at new difficulty
- `bridgeStyle` — bridging lesson generation
