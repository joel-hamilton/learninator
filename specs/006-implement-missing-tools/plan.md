# Implementation Plan: Implement Missing AI Tools

**Branch**: `` | **Date**: 2026-06-18 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/006-implement-missing-tools/spec.md`

## Summary

Two AI tools (`list_feedback_history` and `regenerate_lesson`) are defined in `TEACHER_TOOLS` (`src/ai/teacher.ts`) but lack handler implementations in `src/ai/tools.ts` and the corresponding store methods in `src/db/store.ts`. This causes `"Unknown tool: <name>"` errors when the AI tries to use them. Fix by adding store methods to both `DrizzleMissionStore` and `InMemoryMissionStore`, implementing the two handler functions, registering them in `buildHandlerMap()` and `TOOL_DISPLAY_NAMES`, and adding tests.

## Phase 1: Design

### Data Model

No new entities. The existing `lessons` table has all needed columns:

| Column | Type | Used By |
|--------|------|---------|
| missionId | integer | Both tools (WHERE clause) |
| number | integer | Both tools (WHERE + return) |
| subNumber | integer (nullable) | Both tools (WHERE + return) |
| title | string | Both tools (return + update) |
| slug | string | regenerate_lesson (update) |
| htmlContent | string | regenerate_lesson (update) |
| status | string | listLessonFeedback (return) |
| feedbackRating | string (nullable) | listLessonFeedback (return) |
| feedbackText | string (nullable) | listLessonFeedback (return) |

### Store Interface Additions

```typescript
// Returns lesson summaries including feedback fields
listLessonFeedback(missionId: number): Promise<LessonFeedbackSummary[]>

// Updates lesson title, slug, and HTML content in-place
updateLessonContent(missionId: number, number: number, subNumber: number | null, title: string, slug: string, htmlContent: string): Promise<void>
```

Where `LessonFeedbackSummary` is a new type:

```typescript
type LessonFeedbackSummary = Pick<LessonRow, "number" | "subNumber" | "title" | "status" | "feedbackRating" | "feedbackText">;
```

### Tool Handlers

1. **listFeedbackHistory** — Calls `store.listLessonFeedback(missionId)`, formats result as:
   ```
   Lesson 0001: "Title" — just_right
     Feedback: "text here"
   ```
   Returns "No feedback yet." when no lessons exist.

2. **regenerateLesson** — Validates lesson exists via `store.getLesson()`, calls `store.updateLessonContent(...)`, returns: `Regenerated lesson 0001: "New Title"`. Returns error message if lesson not found.

### Contracts

No external contracts. Tool input schemas are already defined in `src/ai/teacher.ts`.
